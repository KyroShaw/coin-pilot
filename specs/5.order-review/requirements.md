# F-005 已平仓订单复盘 — 需求规格

## 概述

已平仓订单复盘是 coin-pilot 的核心转化路径之一。系统通过用户已绑定的只读 Binance API Key 拉取近 90 天的已平仓订单，以列表形式展示关键交易信息（交易对、方向、开仓价、平仓价、盈亏），并允许用户为每笔订单补充「开仓逻辑」与「平仓逻辑」。在此基础上，由 AI（Claude Opus）结合用户输入的主观交易逻辑与客观成交数据，生成结构化的复盘诊断报告（执行质量、风险控制、改进建议三个维度），报告支持一键复制与 Markdown 导出。

本 feature 旨在帮助交易者把「凭感觉交易」转化为「可被复盘和量化改进的交易」，是产品高价值 AI 推理能力的集中体现。

## 用户故事

- 作为一名已绑定 Binance 只读 API Key 的交易者，我希望系统自动拉取我近 90 天的已平仓订单，这样我不必手动整理交易记录就能开始复盘。
- 作为交易者，我希望在一个清晰的列表里看到每笔已平仓订单的交易对、方向、开仓价、平仓价和盈亏，这样我能快速识别盈利与亏损的交易。
- 作为交易者，我希望为每笔订单记录下当时「为什么开仓」和「为什么平仓」，这样我的主观决策依据能被保存并用于后续分析。
- 作为交易者，我希望 AI 结合我的交易逻辑和真实成交数据，给出一份指出优点、问题和改进建议的复盘报告，这样我能明确知道下一步该改进什么。
- 作为交易者，我希望能一键复制或导出复盘报告为 Markdown，这样我能把它存进自己的交易日志或分享给他人。

## 功能需求

- **[F-005-1] 拉取近 90 天已平仓订单**：通过用户已绑定的只读 Binance API Key（解密后于服务端内存中短暂使用）调用 Binance API，拉取近 90 天的已平仓订单。需处理分页/时间窗口分片，确保订单数量与 Binance 一致；拉取失败时返回友好错误提示。
- **[F-005-2] 订单列表展示**：将拉取到的订单入库后，以列表展示每笔订单的交易对（symbol）、方向（side：买入/卖出，多/空）、开仓价（entryPrice）、平仓价（exitPrice）、盈亏（pnl）及平仓时间（closedAt）。列表支持按时间排序，盈利/亏损以颜色区分。
- **[F-005-3] 用户录入交易逻辑**：用户可为每笔订单填写「开仓逻辑」与「平仓逻辑」两段文本，支持保存与再次编辑。逻辑数据按用户与订单维度持久化，仅本人可见可改。
- **[F-005-4] AI 生成复盘诊断报告**：用户对某笔（或某批）订单提交逻辑后，AI 结合主观逻辑与客观成交数据生成复盘诊断报告。报告至少覆盖三个维度：执行质量、风险控制、改进建议（即优点、问题、建议）。报告生成采用流式输出以满足体验与时延要求。
- **[F-005-5] 复盘报告导出**：生成的复盘报告可一键复制到剪贴板，并可导出为 Markdown 文件（`.md`）。

## 非功能需求

- **AI 报告时延**：复盘报告从用户提交逻辑到生成完成必须 **< 30s**；通过 Claude Opus 流式输出实现「边生成边展示」，避免请求超时并满足体验目标。
- **订单一致性**：拉取的近 90 天已平仓订单数量与 Binance 实际记录一致，不漏单、不重复（入库以订单唯一标识去重幂等）。
- **普通页面性能**：订单列表等普通页面加载 < 2s。
- **错误处理**：Binance 拉取失败、API Key 失效/解密失败、AI 生成失败等场景，均向用户展示友好、可操作的提示，不暴露内部错误细节或敏感数据。
- **安全**：所有复盘相关接口均为 `protectedProcedure`；API Key 解密仅在服务端内存中短暂使用，绝不出现在响应、日志或报告内容中；用户数据严格按 `ctx.session.user.id` 隔离。

## 验收标准

- [ ] **[AC-005-1]** 成功拉取近 90 天已平仓订单，订单数量与 Binance 一致（不漏单、不重复）。
- [ ] **[AC-005-2]** 用户提交交易逻辑后，**30 秒内**生成复盘报告。
- [ ] **[AC-005-3]** 复盘报告包含至少 3 个维度：执行质量、风险控制、改进建议。
- [ ] **[AC-005-4]** 复盘报告可一键复制 / 导出为 Markdown。

## 依赖

- **F-001（强依赖）**：用户账号体系与已绑定的只读 Binance API Key；复用 F-001 提供的加密凭据解密能力（`getBinanceCredential` / 解密工具），在服务端内存中获取明文 Key 用于签名调用。
- **Binance 已平仓订单 API（只读）**：用于拉取近 90 天成交/订单数据，需 HMAC SHA256 签名。
- **Claude AI**：经 `@anthropic-ai/sdk` 调用 `claude-opus-4-8`，配合自适应思考与高 effort、流式输出生成复盘报告；`ANTHROPIC_API_KEY` 经 `@coin-pilot/env` 注入。
- **被复用关系**：F-006 资金曲线将复用本 feature 拉取并入库的已平仓订单数据，数据模型需为此预留（见 design.md）。

## 开放问题

- 「已平仓订单」在**合约**与**现货**下的口径不同：现货无显式「平仓」概念（需以买入/卖出配对推导持仓盈亏），合约有明确的开/平仓与已实现盈亏（realized PnL）。v1 需确认优先支持的交易类型口径——倾向优先支持**合约 USDT 永续**（有明确 realized PnL 与开/平仓语义），现货配对推导留待后续迭代。除此口径问题外，无其它重大开放问题。
