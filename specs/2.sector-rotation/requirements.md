# 板块轮动感知 — 需求规格

## 概述

板块轮动感知功能（F-002）旨在展示当前加密货币市场的热门板块及各板块龙头币种，帮助用户在海量币种中快速定位资金正在流入的赛道和交易机会。系统通过 Binance 公开行情 API 采集行情数据，借助 Claude AI 对当前市场进行板块分类与热度汇总，并以定时刷新（不超过 30 分钟）的方式保证数据的时效性。用户登录后即可在首页查看热门板块列表、热度排名以及每个板块对应的龙头币种价格与涨跌幅。

## 项目信息

- 项目名：`coin-pilot`
- 架构类型：pnpm + Turborepo TypeScript monorepo

## 用户故事

- 作为一名活跃的加密交易者，我希望在首页看到当前热门的板块及其热度排名，以便快速判断市场资金正在流向哪些赛道。
- 作为一名希望挖掘机会的用户，我希望每个板块都能展示其龙头币种的价格与涨跌幅，以便我直接定位到具体的交易标的。
- 作为一名关注时效的用户，我希望看到数据的更新时间，并且数据能够自动定时刷新，以便我相信所看到的行情是新鲜的。
- 作为一名遇到接口异常的用户，我希望在数据拉取失败时看到友好的提示和上次更新时间，以便我了解当前数据状态而不是看到空白或报错。

## 功能需求

- **[F-002-1] 热门板块列表与热度排名**：系统展示当前加密市场的热门板块列表（板块的归类与热度由 Claude AI 汇总生成），并对板块按热度进行排名展示。热度由板块内成分币种的综合表现（如平均涨跌幅、成交量变化）结合 AI 判断得出。
- **[F-002-2] 板块龙头币种**：每个板块展示其代表性龙头币种，包含币种名称/交易对、当前价格、24 小时涨跌幅等关键行情字段。龙头币种行情来自 Binance 公开行情 REST API。
- **[F-002-3] 数据定时刷新**：板块数据与龙头行情通过服务端定时调度机制自动刷新，刷新周期不超过 30 分钟，刷新结果落库并对外提供统一的「最近更新时间」。

## 非功能需求

- **性能**：普通页面响应时间 < 2s；首页板块数据优先读取数据库缓存的快照，不在用户请求时同步调用 Binance 或 Claude。
- **刷新周期**：行情类数据刷新周期 ≤ 30 分钟，由服务端定时任务驱动。
- **错误处理**：当 Binance 或 Claude API 拉取失败时，保留并展示上一次成功的快照数据，同时向用户展示友好提示和「上次更新时间」；不得因单次刷新失败而清空已有数据。
- **鉴权**：访问板块数据的 tRPC procedure 需用户登录（`protectedProcedure`），依赖 F-001 提供的会话。
- **数据一致性**：同一次刷新生成的板块列表与其龙头币种应属于同一快照版本，避免出现板块与龙头时间错配。

## 验收标准

- [ ] **[AC-002-1]** 首页展示至少 5 个热门板块，每个板块至少包含 1 个龙头币种。
- [ ] **[AC-002-2]** 板块数据明确标注更新时间（最近一次成功刷新的时间）。
- [ ] **[AC-002-3]** 数据刷新周期 ≤ 30 分钟，可通过定时调度配置与日志验证。

## 依赖

- **Binance 公开行情 API**：用于获取龙头币种的价格、涨跌幅、成交量等行情数据（只读，使用公开 REST 接口，不强依赖用户 API Key）。
- **Claude AI**：经 `@anthropic-ai/sdk` 调用，用于板块归类与热度「AI 汇总」生成；密钥经 `@coin-pilot/env` 注入。
- **定时调度机制**：服务端定时任务（node 定时器或外部 cron 触发的 HTTP/tRPC 端点），驱动周期性刷新。
- **F-001（账号/登录）**：提供用户会话，板块数据接口需鉴权访问。

## 开放问题

- **板块归类标准的数据来源**：当前方案以 Claude AI 基于行情与币种语义进行归类，缺乏权威的「币种 → 板块」映射数据源。是否需要引入第三方板块分类数据（如 CoinGecko categories）作为辅助输入以提升归类准确性与稳定性，待确认。
- **热度算法的量化口径**：热度是否需要固定的可解释量化公式（如成交量加权涨跌幅），还是完全交由 AI 综合判断，待产品确认。
